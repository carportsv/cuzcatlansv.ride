🔧 Fix OAuth Client - Solución Cliente OAuth Eliminado

❌ Problema Identificado

Error 401: deleted_client

"The OAuth client was deleted"

El cliente OAuth configurado en Firebase/Google Cloud ha sido eliminado o está inválido.

⚠️ Importante

Este NO es un problema de nuestro código. Es un problema de configuración en la consola de Google Cloud/Firebase.

El popup se abre correctamente, pero Google rechaza la autenticación porque el cliente OAuth no existe.

📋 Pasos para solucionarlo:

Paso 1: Ir a Google Cloud Console

Ve a: https://console.cloud.google.com/

Selecciona tu proyecto: taxi-zkt-7f276

Paso 2: Navegar a APIs & Services

En el menú lateral, ve a:

APIs & Services > Credentials

Paso 3: Crear nuevo OAuth Client ID

Haz clic en "+ CREATE CREDENTIALS" y selecciona "OAuth client ID"

Si te pide configurar la pantalla de consentimiento, sigue los pasos básicos.

Paso 4: Configurar el cliente OAuth

Tipo de aplicación: Web application

Nombre: Cuzcatlansv Ride Web

Orígenes autorizados de JavaScript:

http://localhost:8000 http://127.0.0.1:8000 https://tu-usuario.github.io

URI de redirección autorizados:

http://localhost:8000 http://127.0.0.1:8000 https://tu-usuario.github.io

Paso 5: Obtener el nuevo Client ID

Después de crear el cliente, copia el Client ID que se genera.

Paso 6: Actualizar Firebase

Ve a: https://console.firebase.google.com/

Selecciona tu proyecto: taxi-zkt-7f276

Ve a: Authentication > Sign-in method > Google

Haz clic en "Edit" y actualiza el Web SDK configuration con el nuevo Client ID.

1. Recarga la aplicación: http://localhost:8000/

2. Prueba Google Sign-In nuevamente

3. El error "deleted_client" debería desaparecer
