⚠️ Configuración Actual Incorrecta

Basándome en las imágenes que compartiste:

  • ❌ Solo https://auth.expo.io está autorizado
  • ❌ Falta http://localhost:8000 para desarrollo local
  • ❌ Falta http://127.0.0.1:8000 para desarrollo local

📋 Pasos Específicos para Solucionarlo:

Paso 1: Ir a Google Cloud Console

Ve a: https://console.cloud.google.com/

Selecciona tu proyecto: taxi-zkt

Paso 2: Navegar a Credenciales

En el menú lateral, ve a:

APIs & Services > Credentials

Paso 3: Buscar el Cliente OAuth

Busca el cliente: "Expo Auth Web"

ID: 570692523770-bi0bs3j5fadlboh8ahqc0j7k7jr662t7.apps.googleusercontent.com

Si aparece como "eliminado" o "inactivo":
  • Intenta hacer clic en "Restaurar" si está disponible
  • Si no puedes restaurarlo, necesitas crear uno nuevo

Paso 4: Crear Nuevo Cliente OAuth (si es necesario)

Si no puedes restaurar el cliente existente:

  1. Haz clic en "+ CREATE CREDENTIALS"
  2. Selecciona "OAuth client ID"
  3. Tipo de aplicación: "Web application"
  4. Nombre: "Cuzcatlansv Ride Web"

Paso 5: Configurar Orígenes Autorizados

Agrega estos orígenes:

https://auth.expo.io http://localhost:8000 http://127.0.0.1:8000 http://localhost http://127.0.0.1 https://carportsv.github.io https://tu-usuario.github.io
Importante: Mantén https://auth.expo.io si usas Expo, pero agrega los de localhost para desarrollo local.

Paso 6: Configurar URIs de Redirección

Agrega estos URIs:

https://auth.expo.io/@carposv/sv-tazkt http://localhost:8000 http://127.0.0.1:8000 http://localhost http://127.0.0.1 https://carportsv.github.io https://tu-usuario.github.io

Paso 7: Obtener el Nuevo Client ID

Después de guardar, copia el nuevo Client ID que se genera.

Paso 8: Actualizar Firebase

Ve a: https://console.firebase.google.com/

Selecciona tu proyecto: taxi-zkt-7f276

Ve a: Authentication > Sign-in method > Google

Haz clic en "Edit" y actualiza el Web SDK configuration con el nuevo Client ID.

✅ Después de completar los pasos:

1. Recarga la aplicación: http://localhost:8000/

2. Prueba Google Sign-In nuevamente

3. El error "deleted_client" debería desaparecer

4. La autenticación debería funcionar tanto en localhost como en GitHub Pages
