🚨 SOLUCIÓN DEFINITIVA - Crear Nuevo Cliente OAuth

❌ Problema Confirmado

El cliente OAuth actual está marcado como "eliminado" en el sistema de Google

Error: Error 401: deleted_client

Esto significa que Google ha eliminado internamente el cliente, aunque aparezca como "habilitado" en la interfaz.

🎯 Solución: Crear un NUEVO Cliente OAuth

Necesitas crear un cliente OAuth completamente nuevo para reemplazar el que está eliminado.

📋 Pasos para Crear el Nuevo Cliente OAuth:

Paso 1: Ir a Google Cloud Console

Ve a: https://console.cloud.google.com/

Selecciona tu proyecto: taxi-zkt

Paso 2: Navegar a Credenciales

En el menú lateral, ve a:

APIs & Services > Credentials

Paso 3: Crear Nuevo Cliente OAuth

  1. Haz clic en "+ CREATE CREDENTIALS"
  2. Selecciona "OAuth client ID"
  3. Tipo de aplicación: "Web application"
  4. Nombre: "Cuzcatlansv Ride Web App"

Paso 4: Configurar Orígenes Autorizados

En la sección "Authorized JavaScript origins", agrega:

https://auth.expo.io http://localhost:8000 http://127.0.0.1:8000 http://localhost http://127.0.0.1 https://carportsv.github.io https://tu-usuario.github.io

Paso 5: Configurar URIs de Redirección

En la sección "Authorized redirect URIs", agrega:

https://auth.expo.io/@carposv/sv-tazkt http://localhost:8000 http://127.0.0.1:8000 http://localhost http://127.0.0.1 https://carportsv.github.io https://tu-usuario.github.io

Paso 6: Crear y Copiar el Nuevo Client ID

  1. Haz clic en "CREATE"
  2. Se generará un nuevo Client ID
  3. COPIA el nuevo Client ID (será diferente al anterior)

Paso 7: Actualizar Firebase Authentication

Ve a: https://console.firebase.google.com/

Selecciona tu proyecto: taxi-zkt-7f276

Ve a: Authentication > Sign-in method > Google

Haz clic en "Edit"

En "Web SDK configuration", pega el nuevo Client ID

Haz clic en "Save"

✅ Después de completar los pasos:

  1. Recarga la aplicación: http://localhost:8000/
  2. Prueba Google Sign-In
  3. El error "deleted_client" debería desaparecer completamente
  4. La autenticación debería funcionar correctamente

⚠️ Notas Importantes:

  • El cliente anterior está eliminado - no intentes restaurarlo
  • El nuevo Client ID será diferente al anterior
  • Debes actualizar Firebase con el nuevo Client ID
  • Esto es una solución definitiva - resolverá el problema permanentemente
